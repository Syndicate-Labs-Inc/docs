---
title: "References and context"
description: "Give agents the right project information without burying them."
---

References are files or folders you attach so agents can use them as context.

References are different from the working folder and **Uploads**. The working folder is where agents read, edit, and run project files. **Uploads** are files or images attached through chat. References are durable context you intentionally attach at global, team, or agent scope.

## Why references matter

Team agents and solo agents work better when they can see the right information:

- Product requirements.
- README files.
- Architecture notes.
- API schemas.
- Design specs.
- Example files.

## Reference scopes

| Scope | Visibility | Example |
| --- | --- | --- |
| Global | Available across workspaces | Company style guide |
| Team | Available to all agents in a team workspace | Project architecture doc |
| Agent | Available to one team agent or solo agent | Files for a specific task |

## Keep context focused

More context is not always better. Large or irrelevant references can distract agents and increase prompt size.

Use the smallest reference set that helps the agent complete the task.

## Context budgets

Syndicate estimates context size and can trim lower-priority sections when an agent run gets too large. Keep references concise so the most important context survives.

Open **Agent Settings** when you need to tune context for a specific agent. You can adjust history lookback, Agent Comms lookback, reference limits, token budget, and which sections the agent receives, including team context, tasks, Agent Comms messages, references, connectors, and MCP server context. Team context, Agent Comms, and task context apply to team agents; solo agents use their direct chat and attached context instead.

Use prompt analytics to confirm whether the agent is receiving the context you expect before you send a larger task.

## Verify context

If an agent seems to miss important context:

1. Confirm the project is linked to the right working folder.
2. Check whether the material belongs in **Files**, **Uploads**, or references.
3. Attach durable docs as references at the narrowest useful scope.
4. Send a small task that asks the agent to summarize or apply the reference.
