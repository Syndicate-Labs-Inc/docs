---
title: "References and context"
description: "Give agents the right files and folders as context, scoped globally, per team, or per agent."
---

References are files and folders you attach as read-only context. Their content is injected into the agent's prompt on every run, so the agent does not have to go find it.

## Three scopes

| Scope | Visible to | Managed in | Use for |
|---|---|---|---|
| **Global** | All agents, all workspaces | Settings | Company standards, style guides, shared docs |
| **Team** | All agents in one team | Team settings | Project README, schema, requirements |
| **Agent** | One agent | Agent configuration | Role-specific source material |

## How references are injected

On each dispatch, Syndicate resolves the agent's references and inlines them:

- **Files**: content is included up to 64 KB per file, within a 200 KB total budget. Larger files contribute path and metadata only.
- **Folders**: a manifest of file names and sizes (not the contents).
- **PDFs**: extracted text (up to 10 MB source size).

References can live anywhere on disk; unlike the working directory they are context, not a workspace. Agents read them but cannot edit them through the reference mechanism.

## Adding and removing

Click the **+** next to the relevant references section (sidebar, team settings, or agent config), then pick files or folders in the native picker. Remove a reference the same place you added it.

Click any reference to open a read-only preview with line numbers and a language badge.

## Keep context lean

More context is not better; it competes with conversation history and inbox messages for the agent's attention and token budget.

- Attach the few files that actually define the task, not the whole repository (the agent can read workspace files itself with its tools).
- Prefer team scope for shared project facts, agent scope for role-specific material, and global scope sparingly.
- Use the agent's **Context** settings and **Prompt Analytics** to see how many characters references consume per run, and trim what is not earning its place. See [Agents](/features/agents#context).
