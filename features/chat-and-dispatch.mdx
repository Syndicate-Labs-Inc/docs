Chat is the direct way to work with an agent. Dispatch is how Syndicate starts agent runs, routes inter-agent messages, streams progress, and wakes specialists when Atlas assigns work.

## Direct chat

Use direct chat when you want to talk to the selected agent.

1. Open a team or solo workspace.
2. Select an agent.
3. Type a clear request.
4. Attach files or images if needed.
5. Send the message.

The agent receives your message, recent conversation, its instructions, team context, tasks, inbox context, and available references based on its context settings.

## Attachments

The chat input supports:

- Picking files from your computer.
- Dragging files onto the input.
- Pasting images.
- Previewing image thumbnails before you send.
- Removing attachments before send.

Files attached through team chat can appear later in the **Uploads** tab in the bottom tray.

## Slash commands

Agents can expose slash commands from installed skills. When a slash command menu appears, choose a skill to insert `/{skill-name}` into the chat input.

Slash commands are scoped to the selected agent's specialization and installed skills.

## Agent Comms

**Agent Comms** shows inter-agent dispatch messages. It keeps coordination traffic separate from your direct chat.

For a specialist, **Agent Comms** shows messages received from Atlas or other agents and messages sent by that specialist. For Atlas, it shows the broader team dispatch stream.

Use **Agent Comms** when you want to inspect what Atlas asked a specialist to do, what the specialist reported back, or why a handoff happened.

## Manual dispatch

Manual dispatch starts an agent run on demand. Use it when you want an agent to check messages, continue a task, review work, or run a specific request.

## Automatic dispatch

Automatic dispatch can happen when an idle agent receives a message from Atlas or another agent. This lets teams continue a workflow without you manually waking each specialist.

Automatic dispatch does not run while the team is hard paused. A team can also soft pause while a **User Task** waits for your input.

## What you see while an agent runs

During a run, Syndicate can show:

- Agent status.
- Streaming response text.
- Tool activity.
- Thought or progress summaries.
- Files, commands, and changes detected during the run.
- Context and activity events in the **Activity** tab.

## Running agent controls

When an agent is running, the chat input can show:

- **Answer Now**: interrupt the current run and ask the agent to respond with what it has so far.
- **Stop**: cancel the current response for that agent.
- **Send** or **Steer**: send a new message, including guidance for the active run when supported.

Stopping one agent response is different from **Stop Team**. Use **Stop Team** when you need a hard pause across the whole team.

## Manager operations

When Atlas is selected, use the right rail tabs for team operations:

- **User Tasks** for decisions, approvals, blockers, and interruption prompts.
- **Agent Tasks** for tracked work.
- **Task Health** for risk and task quality signals.

User approval and decision work should route through **User Tasks**, not chat approval cards.
