title: "Settings"
description: "Manage account, general app behavior, providers, MCP servers, teams, and agents."
---

Settings is the control center for app-level configuration. Team and agent settings live inside the workspace so you can adjust them in context.

## Account

Use **Account** settings to view your signed-in Syndicate account and sign out.

## General

Use **General** settings to manage:

- App data path.
- Default provider.
- Default model.
- Theme.

## Providers

Use **Providers** settings to:

- Connect providers.
- Disconnect providers.
- Refresh provider state.
- See installed and authenticated status.
- Choose defaults.

Agents can only run on models from connected providers.

## MCP

Use **MCP** settings to:

- Review available MCP catalog entries.
- See setup requirements.
- Install or configure an MCP server.
- Add an MCP server manually.

Installing an MCP server does not give every agent access. Enable MCP servers for the agents that need them from **Agent Settings**.

## Team settings

Open **Team Settings** from **Team Hub**, a team card, or the **Team Workspace** header.

Use **Team Settings** to:

- Rename the team.
- Review or relink the working directory.
- Set **Interruption Level**.
- Hard pause or resume the team.
- Delete the team.

**Interruption Level** controls when Atlas asks for input. **Hard pause** stops current team dispatches and blocks new automatic dispatch. Resuming a hard-paused team does not restart agents automatically.

## Agent settings

Open **Agent Settings** from the right rail in **Team Workspace** or from a solo agent.

Use **Agent Settings** to:

- Review identity and specialization.
- Rename non-manager agents.
- Read the agent's instructions or **Soul**.
- Change context settings.
- Review skills and slash commands.
- Enable or disable MCP servers for that agent.
- Delete a non-manager agent after confirmation.

Context settings control how much history, Agent Comms context, tasks, team context, references, and connector context the agent receives during runs. Use the presets when you want a quick choice, then tune history lookback, Agent Comms lookback, reference limits, token budget, and included context sections when a run needs more focus. Team context, Agent Comms, and task context apply to team agents; solo agents use their direct chat and attached context instead.

Prompt analytics help you see the estimated size of the prompt, how much history is included, and which context sections are using the budget.
