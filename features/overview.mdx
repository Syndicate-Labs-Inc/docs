## How a team coordinates work

A **Manager** is the agent that coordinates a team. Instead of you assigning each step, you give the Manager an outcome and it decides who on the team should do what.

**Dispatch** is how Syndicate hands a message or task to an agent so it actually starts running. The Manager dispatches work to specialists, specialists dispatch results and follow-ups back, and you stay in the loop through the inbox.

```mermaid
flowchart LR
  User["You"] --> Workspace["Workspace"]
  Workspace --> Team["Team"]
  Workspace --> Solo["Solo agent"]
  Team --> Manager["Manager"]
  Manager --> Agents["Specialist agents"]
  Agents --> Tasks["Tasks"]
  Agents --> Inbox["Inbox messages"]
  Agents --> Tools["Skills and MCP servers"]
  Agents --> Project["Local project folder"]
  Solo --> Tasks
  Solo --> Inbox
  Solo --> Tools
  Solo --> Project
```

## Common ways to use Syndicate

| Goal | Use |
| --- | --- |
| Fix a bug | Solo engineer or team with Manager, Coder, and QA |
| Build a feature | Team with Manager plus specialists |
| Review code | Code review team or QA specialist |
| Write docs | Technical writer agent with project references |
| Research a topic | Researcher or research team |
| Coordinate many tasks | Team workspace with Manager control modes |
