An agent is a single AI worker. This page covers its anatomy and the configuration panel where you shape it.

## Anatomy of an agent

| Part | What it is |
|---|---|
| **Name** | The display name you choose (for example "Schema Wrangler") |
| **Specialization** | Its role and baked-in skill set, chosen from the [Marketplace](/guides/agent-marketplace) at creation |
| **Model** | The LLM powering it, from any connected provider |
| **Soul** | The markdown prompt defining its expertise, personality, and workflow |
| **Skills** | Slash-command workflows resolved from its specialization |
| **Tools (MCP)** | External integrations toggled on per agent |
| **Context settings** | How much history and inbox material it sees per run |
| **Permissions** | What it may do without asking (see [Permissions and safety](/reference/permissions-and-safety)) |

## Agent statuses

Agents show a live status everywhere they appear (sidebar, canvas, headers):

- **Active** (green): currently dispatched and working
- **Idle** (gray): waiting for work
- **Waiting** (amber): needs approval or input, often with a short reason
- **Blocked** (red): hit an error; open its chat to see what happened

## The agent configuration panel

Open it from the agent's settings icon. Sections:

### Identity

Name, specialization badge, and the model selector. Models are listed per provider with friendly names (for example "Sonnet 4.6", "GPT-5.3 Codex", "Gemini 2.5 Pro").

### Soul

The agent's system prompt as markdown. Built-in specializations are read-only; user-made agents have an **Edit** button for the editable section.

<Warning>
Saved Soul edits cannot be undone automatically. Copy the current text somewhere safe before large rewrites.
</Warning>

### Context

Controls what the agent sees on each run. Presets: **Lean**, **Balanced**, **Rich**, or **Custom**. The editor exposes:

- **History & Memory**: message lookback (0-200), max characters per message, and a role filter.
- **Scope & Budget**: inbox lookback, reference max, and a token budget (0 = unlimited).
- **Sections**: toggle which context blocks are injected (Team, Connectors, Tasks, Inbox, References).
- **Prompt Analytics**: after a run, see prompt size, estimated tokens, and a per-section breakdown so you can trim what is not earning its keep.

### Skills

The list of slash-command skills this agent carries, resolved from its specialization. See [Skills](/features/skills).

### Tools (MCP)

Per-agent toggles for every MCP server installed in Settings, each with a health dot. If the list is empty: "No MCP servers installed. Add them in Settings > MCP, then turn them on here." See [MCP servers](/features/mcp).

### Danger Zone

**Delete Agent** removes the agent and all its configuration, with a two-step confirmation.

## Permission mode

Each agent has a permission mode, set from the shield selector:

| Mode | Label | Behavior |
|---|---|---|
| Ask first | "Shows what the agent wants to do before executing" | Every tool use prompts you |
| Auto-accept edits | "Agent can read & edit files automatically" | File operations run automatically, commands still prompt |
| Full auto | "Agent can run commands, edit files, everything" | No prompts |

Details and recommendations: [Permissions and safety](/reference/permissions-and-safety).
