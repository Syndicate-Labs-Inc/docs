A solo agent is a one-on-one workspace with a single specialist. There is no Manager, no **Agent Comms**, no inter-agent dispatch, and no roster to coordinate. It is just you, one agent, and a project folder.

Use solo agents when you know which specialist you need and want fast back-and-forth instead of orchestration.

## When to use a solo agent

Use a solo agent when:

- The task is focused.
- You know which specialist you need.
- You want fast back-and-forth.
- The work does not need delegation.
- You do not need **User Tasks**, **Agent Tasks**, or **Task Health**.

## Good solo agent examples

| Agent | Example task |
| --- | --- |
| Frontend Developer | Polish a settings screen |
| Technical Writer | Write a tutorial page |
| QA Engineer | Create a regression checklist |
| Researcher | Compare product options |
| Backend Developer | Review an API route |

## Switching from solo to team

If a solo workflow grows into multiple parallel tasks, create a team and add specialists. Solo agents are best for focus; teams are best for coordination. See [Teams](/features/teams) for **Interruption Level**, **Agent Comms**, **User Tasks**, and team-creation paths.
