---
title: "Providers and models"
description: "Which providers Syndicate supports, the models they offer, and how model selection works."
---

Syndicate does not ship its own models. Agents run on the provider accounts you connect: Claude, OpenAI (Codex), and Google Gemini. Connect one or all three; agents in the same team can run on different providers.

## Supported providers

| Provider | Account used | Auth flow |
|---|---|---|
| **Claude** | Your Claude account | Browser sign-in, paste a code back into the app |
| **OpenAI (Codex)** | Your ChatGPT account (with Codex access) | CLI sign-in |
| **Google Gemini** | Your Google account | CLI sign-in |

Each provider works through its CLI. If the CLI is missing, Syndicate offers an **Install CLI** button and installs it using its bundled runtime, so you do not need Node.js set up yourself. Connection details and troubleshooting: [Connect providers](/guides/connect-providers).

## Available models

Models appear only for providers that are connected. Current lineup:

| Provider | Models |
|---|---|
| Claude | Fable 5, Opus 4.8, Opus 4.7, Opus 4.6, Sonnet 4.6, Haiku 4.5 |
| OpenAI (Codex) | GPT-5.5, GPT-5.4, GPT-5.3 Codex |
| Google Gemini | Gemini 2.5 Pro, Gemini 2.5 Flash, Gemini 3 Flash Preview |

Model lists update as providers ship new versions; the in-app selector always reflects what your connected CLIs support.

## How model selection works

There are two levels:

1. **Default model** (Settings, Providers tab, "Default Model For New Agents"): used for new teams and solo agents. Existing agents keep their current model.
2. **Per-agent model**: set in the team creation wizard or any time in the agent's configuration panel. Each agent can use a different model.

If a provider disconnects, agents pointing at its models are automatically resolved to the first available model from your remaining providers, so nothing gets stuck.

## Choosing models for a team

A pattern that works well:

- **Atlas (Manager)**: your most capable model (for example Fable 5). Planning and coordination benefit most from reasoning quality.
- **Coders**: a strong code model (for example GPT-5.3 Codex or Sonnet 4.6).
- **Reviewers, writers, support roles**: a balanced model (Sonnet 4.6).
- **High-volume, low-stakes monitors**: a fast model (Haiku 4.5).

The built-in [team templates](/reference/team-templates) follow this pattern out of the box.

## Disconnecting and reconnecting

Manage providers in Settings under **Providers**: each card shows a **Connected** or **Not connected** badge, the CLI version, and its model list, with **Connect**, **Reconnect**, and **Disconnect** actions. **Reconnect** lets you switch accounts.

If credentials expire mid-session, an amber banner appears with a one-click **Reconnect** action.
