Tasks help you see what a team is doing, what needs you, and which active work may be risky or blocked.

In Team Workspace, task work is split across three Atlas tabs:

- **User Tasks**: decisions, approvals, blockers, and interruption prompts that need your input.
- **Agent Tasks**: tracked work assigned to Atlas or specialists.
- **Task Health**: risk and quality signals for active tasks.

## User Tasks

**User Tasks** replace chat approval cards. Use them when the team needs a human decision.

A **User Task** can come from:

- A blocked task.
- A task that needs human review before completion.
- A message sent to you.
- An Atlas interruption request.

Each item shows the requester, source, impact, description, related task or message, and latest update time.

| Impact | Meaning |
| --- | --- |
| **Blocking** | The team should not continue until you respond |
| **High** | The decision affects scope, risk, or completion |
| **Medium** | The team can continue some work but needs direction |
| **Low** | The item needs review but is not urgent |

Use **Reply** to answer Atlas. Use **Resolve** to close an item without a reply. Use **Allow** or **Dismiss** when Atlas asks you to approve a manager step.

## Agent Tasks

**Agent Tasks** shows real tracked work for Atlas and specialists. It excludes routine dispatch bookkeeping.

Use it to:

- Filter tasks by status.
- Review the title, assignee, risk, phase, and update time.
- See whether a task requires a human decision.
- Cancel one task.
- Abort all visible abortable tasks when the plan needs a reset.

| Status | Meaning |
| --- | --- |
| **Todo** | Work is queued or planned |
| **In Progress** | An agent is actively working |
| **Blocked** | Work needs input or cannot continue |
| **Done** | Work is complete |
| **Canceled** | You canceled the task |

When you cancel a task, Atlas gets a receipt and guidance not to recreate or reassign that task unless you explicitly ask.

## Task Health

**Task Health** is a watchlist, not a second task board. Use it to spot task risk before the team loops or stalls.

Task Health can flag:

- Blocked tasks.
- Tasks requiring human decisions.
- High-risk or medium-risk tasks.
- Unassigned tasks.
- Tasks with weak deliverables, acceptance criteria, or done conditions.

Click **Nudge Manager** on a specific risk item when you want Atlas to address that item. Syndicate sends Atlas the task, severity, reason, and requested next step.

## Assign Task

Click **Assign Task** from the right rail when you want to create tracked work directly.

The form includes:

- **Agent**.
- **Title**.
- **Instructions**.
- **Deliverable**.
- **Done Condition**.
- **Acceptance Criteria**.
- **Phase**.
- **Risk**.
- **Requires human decision before completion**.
- **Notify and start agent**.

Choose **Notify and start agent** when you want the assignee to wake and start through normal dispatch. If the task requires a human decision, Syndicate routes that review through **User Tasks**.

## Best practices

- Keep tasks outcome-focused.
- Assign tasks to the agent best suited for the work.
- Add a deliverable, done condition, and acceptance criteria.
- Use **User Tasks** for decisions instead of burying approvals in chat.
- Use **Task Health** to catch risk early.
