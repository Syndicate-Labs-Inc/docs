MCP servers extend what agents can access. They can connect agents to services such as documents, files, internal tools, or other systems.

{/* Screenshot: Settings > MCP catalog page, showing at least one ready entry and one docs-only entry so the distinction in the text below is visible. */}

## Where MCP lives

Open **Settings > MCP** to view available MCP servers and setup requirements.

The current MCP catalog includes ready entries and docs-only entries. Ready entries can be configured from the app. Docs-only entries link out while the integration package is still being verified.

{/* Screenshot: agent configuration panel with the MCP servers section showing one enabled and one disabled server toggle. */}

## Agent-level access

Installing or configuring an MCP server does not mean every agent should use it. Enable MCP servers only for the agents that need them.

## Good MCP use cases

- Let a researcher use an approved source material connector.
- Let a writer use configured documentation sources.
- Let an agent use a project-specific MCP server that you explicitly enabled.

## Safety tips

- Start with read-only capabilities where possible.
- Avoid giving broad external access to every agent.
- Keep credentials out of screenshots and docs.
- Test one server with one agent before rolling it out to a team.
