---
title: "MCP"
description: "Use MCP servers to connect agents to external tools and data."
---

MCP servers extend what agents can access. They can connect agents to services such as documents, files, internal tools, or other systems.

## Where MCP lives

Open **Settings > MCP** to view available MCP servers and setup requirements.

The current MCP catalog includes entries such as Google Drive and Notion, with setup instructions and required environment variables.

## Agent-level access

Installing or configuring an MCP server does not mean every agent should use it. Enable MCP servers only for the agents that need them.

## Good MCP use cases

- Let a researcher read shared source material.
- Let a writer pull facts from documentation.
- Let a support agent inspect external knowledge bases.
- Let an engineering agent use project-specific tools.

## Safety tips

- Start with read-only capabilities where possible.
- Avoid giving broad external access to every agent.
- Keep credentials out of screenshots and docs.
- Test one server with one agent before rolling it out to a team.
