A workspace is the place where Syndicate organizes agent work. Most workspaces point at a local project folder.

## Workspace types

| Type | Best for | What it contains |
| --- | --- | --- |
| Team | Coordinated multi-agent work | Manager, specialists, inboxes, tasks |
| Solo agent | Direct one-on-one work | One agent attached to a project folder |

## Project folders

When you create a team or solo agent, choose the local folder where the agent should work. Agents use that folder as their working directory when reading files, editing code, running commands, and gathering context.

<Warning>
  Choose a folder where you are comfortable letting agents operate. Use Git or another backup system before giving agents broad edit permission.
</Warning>

## Team Hub

The Team Hub shows your multi-agent workspaces, connected provider state, recent teams, and active work. Use it when you want to see the bigger picture across teams.

## Solo Hub

Solo agents are designed for focused, direct work. Use a solo agent when coordination overhead would slow you down.

## Changing a working directory

Open team settings, then use **Browse** in the working directory area. Syndicate relinks the workspace to the selected folder.
