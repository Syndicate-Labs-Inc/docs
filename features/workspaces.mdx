A workspace is a team or solo agent plus the local folder it works in. This page covers the two workspace types, the working directory, and the main layout.

## Two workspace types

| | Team | Solo agent |
|---|---|---|
| Agents | Manager (Atlas) + specialists | One specialist, no Manager |
| Coordination | Tasks, delegation, Agent Comms | Direct chat only |
| Best for | Multi-role projects | Quick, focused work |
| Found in sidebar | **Team Hub** | **Solo Agents** |

## The working directory

Every workspace is tied to one local folder, chosen when you create it. Agents read, write, and run commands inside that folder.

- Change it later in **Team Settings** under **Working Directory** (Browse to pick a new folder).
- To give agents a file that lives outside the folder, attach it in chat; attachments appear in the bottom tray's **Uploads** tab.
- With **workspace confinement** enabled (Settings, General), agents are sandboxed at the OS level so they cannot read or write outside their working folder. See [Permissions and safety](/reference/permissions-and-safety).

<Warning>
Agents with edit permissions change real files in the working directory. Use a Git repository or another backup before giving agents broad edit access, so every change is reviewable and reversible.
</Warning>

## Sidebar

The left sidebar is the workspace switcher:

- **Agent Marketplace**: browse and add agent specializations.
- **Team Hub**: your teams, each expandable to show its agents with live status dots (green working, amber waiting, red error, gray idle).
- **Solo Agents**: your standalone agents.
- **Account footer**: your profile, plan, and a shortcut to Settings.

Toggle the sidebar with **Ctrl+B**.

## Inside a team workspace

A team workspace has three main tabs:

- **Dashboard**: the visual canvas. Atlas in the center, agent tiles around it, animated lines showing live message flow, and the three task panels (User Tasks, Agent Tasks, Task Health) on the right.
- **Chat**: your direct conversation with Atlas.
- **Dispatch**: the raw agent-to-agent message feed.

Clicking an agent tile opens that agent's focused view, with **Chat** and **Agent Comms** tabs in the right rail.

The collapsible **bottom tray** holds:

- **Activity**: a chronological log of agent actions (commands run, files changed, summaries).
- **Files**: every file agents have created or modified, with previews.
- **Uploads**: files you have attached in chat for this team, refreshed automatically.

## Workspace lifecycle

- **Create** a workspace from the sidebar (+ buttons) or the command palette (**Ctrl+K**).
- **Pause** a team (Team Settings, "Team Hard Pause", or the card menu) to stop all dispatches. Resuming does not restart agents; they run again only when prompted.
- **Relink** the working folder in Team Settings without losing agents or history.
- **Delete** a team from the Danger Zone in Team Settings. This permanently removes the team and all its agents. Files in the working directory are not deleted.
