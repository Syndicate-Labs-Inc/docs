A team is a shared [workspace](/features/workspaces) for agents working against the same local project folder. Start here when you want a Manager to coordinate specialists.

## Create a team manually

{/* Video: Build Your Team flow from click to completed workspace. */}

<Steps>
  <Step title="Start team creation">
    Click **Create Team**.
  </Step>
  <Step title="Choose manual setup">
    Select **Build Your Team** if you want to start from scratch.
  </Step>
  <Step title="Select a project folder">
    Choose the local folder the team should read, edit, and run commands in.
  </Step>
  <Step title="Review agents">
    Add or review the Manager and specialists before creating the team.
  </Step>
  <Step title="Create the team">
    Finish setup and open the new team workspace.
  </Step>
</Steps>

## Use AI-guided setup

{/* Video: describe a project, review proposed agents, select folder, create team. */}

Use the guided team flow when you want Syndicate to draft a starting roster from a project description.

1. Describe your project.
2. Review the proposed agents.
3. Edit names, roles, or instructions.
4. Select the project folder.
5. Create the team.

## Pick the working folder

{/* Screenshot: folder picker and selected path in setup. */}

The selected folder becomes the working directory for agent runs. Agents use it when they read files, edit code, run commands, and gather project context.

## Choose a Manager control mode

{/* Screenshot: Manager control mode options. */}

| Mode | Use when |
| --- | --- |
| **Autopilot** | You trust the team to keep work moving without checking every step |
| **Approve New Work** | You want the Manager to ask before expanding scope |
| **Approve Each Step** | You want tight control over every delegation |

You can change this later in team settings.

## First task to try

{/* Video: send a first task, show the Manager assigning it to a specialist, then show the specialist beginning work. */}

Ask one agent to inspect the project and summarize:

- What the app does.
- How to run it locally.
- Which files are most important.
- What the agent would improve first.

<Tip>
This gives you a quick sanity check before assigning larger work.
</Tip>
