A team is a group of agents that run against the same working folder and share team context inside Syndicate.

## Create a team manually

1. Click **Create Team**.
2. Choose **Build Your Team** if you want to start from scratch.
3. Choose the local project folder the team should work in.
4. Add or review agents.
5. Create the team.

## Use AI-guided setup

Use the guided team flow when you want Syndicate to draft a starting roster from a project description.

1. Describe your project.
2. Review the proposed agents.
3. Edit names, roles, or instructions.
4. Select the project folder.
5. Create the team.

## What the working folder controls

The selected folder becomes the working directory for agent runs. Agents use it when they read files, edit code, run commands, and gather project context.

## Choose a Manager control mode

If your team has a Manager, choose how much autonomy the Manager should have when delegating or following up with specialists.

| Mode | Use when |
| --- | --- |
| Autopilot | You trust the team to keep work moving without checking every step |
| Approve New Work | You want the Manager to ask before expanding scope |
| Approve Each Step | You want tight control over every delegation |

You can change this later in team settings.

## First task to try

Ask one agent to inspect the project and summarize:

- What the app does.
- How to run it locally.
- Which files are most important.
- What the agent would improve first.

This gives you a quick sanity check before assigning larger work.
