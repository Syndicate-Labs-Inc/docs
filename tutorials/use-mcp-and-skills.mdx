## Add a skill

{/* Video: enable a skill for an agent and show it available in the agent configuration. */}

Skills are reusable instructions for specialized work. Use them when an agent needs a reliable workflow for tasks like documentation, QA, frontend design, analytics, or release management.

<Steps>
  <Step title="Open the agent">
    Choose the agent that should use the skill.
  </Step>
  <Step title="Find skills">
    Open the skills area in the agent configuration.
  </Step>
  <Step title="Add the right skill">
    Choose the skill that matches the agent's job.
  </Step>
  <Step title="Save the agent">
    Return to the team and give the agent a task that uses the skill.
  </Step>
</Steps>

## Add an MCP tool

{/* Video: enable an MCP tool and show connected/available state. */}

MCP servers connect agents to external capabilities only after you install or configure them and enable them for an agent. Depending on your setup, a server may expose files, browser automation, documents, or another service.

<Steps>
  <Step title="Install or configure the MCP server">
    Complete the setup required by that tool.
  </Step>
  <Step title="Enable it for an agent">
    Open the agent configuration and make the tool available.
  </Step>
  <Step title="Confirm access">
    Check that the tool shows as available before asking the agent to use it.
  </Step>
  <Step title="Test with a small task">
    Ask for something simple that clearly requires the tool.
  </Step>
</Steps>

## When to add a capability

{/* Screenshot: agent configuration with a small number of clear capabilities. */}

Add a skill or tool when:

- The same task appears repeatedly.
- The task needs a consistent process.
- The agent needs access to a system outside the app.
- The output quality improves with domain-specific instructions.

## Keep capabilities understandable

Give each capability a clear name and purpose. Agents should not need to guess when to use it.
