## Add an agent

{/* Video: add an agent from an existing team workspace. */}

<Steps>
  <Step title="Open a team">
    Choose the team that should receive the new agent.
  </Step>
  <Step title="Add an agent">
    Click the add agent control.
  </Step>
  <Step title="Name the agent">
    Use a short, specific name that makes the job obvious.
  </Step>
  <Step title="Choose role and model">
    Pick the role, provider, and model that fit the work.
  </Step>
  <Step title="Save the agent">
    Confirm the setup and return to the team roster.
  </Step>
</Steps>

## Write useful instructions

{/* Screenshot: instructions editor with a concise example. */}

Good agent instructions answer:

- What is this agent responsible for?
- What should it avoid changing?
- Which tools or skills should it prefer?
- When should it ask for help?
- How should it report progress?

## Example roles

{/* Screenshot: role picker or a configured roster with several specialists. */}

| Role | Good for |
| --- | --- |
| Frontend engineer | UI changes, layout fixes, component work |
| QA engineer | Test plans, regression checks, browser testing |
| Technical writer | Tutorials, release notes, help docs |
| Researcher | Competitive research, product analysis |
| Release manager | Shipping checklists, changelogs, deployment review |

## Keep teams focused

Start with two or three agents. Add more only when a repeated kind of work needs its own specialist.

## Done when

- The agent appears in the team roster.
- It has a role, provider, model, and clear instructions.
- You know what kind of task should go to that agent.
