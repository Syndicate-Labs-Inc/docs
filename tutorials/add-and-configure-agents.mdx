Agents work best when each one has a clear job.

## Add an agent

1. Open a team.
2. Click the add agent control.
3. Give the agent a short name.
4. Choose a role.
5. Select a provider and model.
6. Save the agent.

## Write useful instructions

Good agent instructions answer:

- What is this agent responsible for?
- What should it avoid changing?
- Which tools or skills should it prefer?
- When should it ask for help?
- How should it report progress?

## Example roles

| Role | Good for |
| --- | --- |
| Frontend engineer | UI changes, layout fixes, component work |
| QA engineer | Test plans, regression checks, browser testing |
| Technical writer | Tutorials, release notes, help docs |
| Researcher | Competitive research, product analysis |
| Release manager | Shipping checklists, changelogs, deployment review |

## Keep teams focused

Start with two or three agents. Add more only when a repeated kind of work needs its own specialist.
