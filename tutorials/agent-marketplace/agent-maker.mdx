Agent Maker is a built-in Syndicate specialist whose job is to draft other agents. Unlike the Manager and the rest of the built-in roster, you don't add Agent Maker from the **Syndicate** category. Instead, you open it through **Add Agent Package → Agent Maker**, because it produces new agent packages rather than joining a workspace itself.

Describe the specialist you want, and Agent Maker generates a Marketplace package — title, description, skills, and workflow guidance — that you can review, revise, and save.

Use it for repeatable roles that aren't in the built-in catalog: launch reviewer, compliance reader, customer research analyst, release-note writer, prompt auditor, QA checklist owner, or any specialist your team uses often.

## How Agent Maker drafts a package

Agent Maker is built and tuned to Syndicate's package conventions. It knows the shape of a Marketplace package — title, description, skills, and workflow guidance — and how those pieces fit together. The saved result is addable to a team or solo workspace without extra cleanup.

It is also tuned to recognize when the role you describe overlaps a built-in specialist. If a built-in already fits, Agent Maker can steer you toward that one instead of generating a duplicate package.

## Agent Maker vs. asking a Manager to create one

A Manager in a regular workspace can advise on what an agent should do, but a Manager conversation produces a chat-shaped answer, not a saved package. Agent Maker is the path that produces a Marketplace package — one that lands in the **User Made** category and is addable to teams or solo workspaces alongside the built-ins.

Use Agent Maker when you want the role to persist and be reused. Ask a Manager when you only need help thinking through what the role should be.

## When to use Agent Maker

Reach for Agent Maker when:

- No built-in specialist matches the role you need.
- The role is specific to your project or team.
- You expect to reuse the agent across multiple workspaces.

If a built-in specialist already covers the role, [use the built-in](/tutorials/agent-marketplace/built-in-syndicate-agents) — it's faster and already battle-tested.

## Before you start

You need:

- At least one connected provider.
- A clear description of the specialist you want.
- A rough idea of what the agent should do repeatedly.

<Tip>
Agent Maker works best when you describe the agent's job, inputs, outputs, boundaries, and what "good work" looks like.
</Tip>

## Open Agent Maker via Add Agent Package

1. Open **Agent Marketplace**.
2. Click **Add Agent Package**.
3. Choose **Agent Maker**.

## Describe the agent

Write a focused description.

Good description:

```txt
A read-only launch review specialist that checks release notes, onboarding copy, screenshots, and setup instructions before a public launch. It should flag unclear language, missing steps, broken links, and claims that are not supported by the app.
```

<Frame>
  ![Make a custom agent modal with the launch review specialist description filled into the "Describe the agent" field, ready to Generate Agent.](/assets/screenshots/agents/agent_marketplace_custom_docs_example.png)
</Frame>

Avoid vague descriptions:

```txt
Make me a good business agent.
```

## Generate the package

1. Enter the description.
2. Click **Generate Agent**.
3. Wait for Syndicate to draft the package.

Syndicate returns a Marketplace package with a starting configuration:

- Agent title.
- Description.
- Skills the agent will use.
- Workflow guidance.

Treat this as a starting configuration — review it before saving.

## Revise before saving

If the draft is close but not perfect:

1. Use the feedback box.
2. Tell Agent Maker what to change.
3. Click **Revise Draft**.

Useful revision prompts:

- "Make it read-only and explicitly say it should not edit files."
- "Add a checklist-style review command."
- "Remove analytics work and focus only on docs."
- "Make this more tactical for release QA."

## Save to Marketplace

When the draft looks right, click **Save to Marketplace**.

The package appears in the **User Made** category. You can add it to a team or create a solo workspace from it.

## Add the custom agent to a workspace

Open the package in the **User Made** category, click **Add to...**, and choose **Existing Team** or **Solo Workspace**. For the full flow and what happens after you add an agent, see [Choose how you want to add an agent](/tutorials/agent-marketplace#choose-how-you-want-to-add-an-agent).

## Skills in your custom agent

Agent Maker bundles skills into the draft based on your description. Skills are reusable workflow instructions — they tell the agent when to use a workflow, what steps to follow, and what output to produce.

After saving the package, open the agent in any workspace to inspect or change those skills in the agent configuration panel. See [Skills](/features/skills) for what skills do and when to add more.

## Edit a user-made package later

1. Open **Agent Marketplace**.
2. Go to **User Made**.
3. Select the package.
4. Click **Edit**.
5. Update or revise the package.

## Limits & gotchas

- **Agent Maker drafts packages — it doesn't run the role for you.** Saving a package puts it in **User Made**. To actually use the agent, add it to a team or solo workspace from there.
- **Draft quality follows description quality.** Vague inputs produce vague agents. Spell out the job, inputs, outputs, boundaries, and what "good work" looks like, then iterate with **Revise Draft**.
- **Agent Maker runs through your connected provider.** Drafting and revising a package uses your provider account the same way any other agent run does — Agent Maker is not free of provider usage.
- **The first draft is a starting configuration, not a finished agent.** Review the skills, instructions, and references before you save, and tighten them in the agent settings after you add it to a workspace.
- **Use a built-in when one already fits.** Agent Maker is for roles that aren't covered by a built-in. If a built-in specialist covers the role, prefer it.
- **Iterating on a user-made agent is normal.** Save the package, try it on a small task, then come back and update the saved package with what you learned.

## Verify it worked

Send a task that matches the custom role.

Example:

```txt
Review this onboarding doc as the launch review specialist. Do not edit files. Return the top issues, why they matter, and exact suggestions.
```

Expected result: the agent responds according to the package you created.
