Providers supply the [models](/features/providers-and-models) agents use to run. Syndicate connects to Claude, OpenAI/Codex, and Gemini through their local CLIs, then verifies that each CLI is installed, signed in, and able to run.

Connect at least one provider to start. Adding more gives each agent a wider choice of models.

Before you start, finish [Install and sign in](/tutorials/install-and-sign-in) so the Syndicate app is open on your machine.

## Pick a provider

Each provider connects through its own CLI and sign-in method. Pick one to start — the steps for each are below.

| Provider | CLI | Sign-in |
| --- | --- | --- |
| Claude | Claude Code CLI | Browser OAuth |
| OpenAI/Codex | Codex CLI | Browser ChatGPT login plus a capability check |
| Gemini | Gemini CLI | In-app terminal plus Google browser sign-in, or a Gemini API key |

<Note>
**What's a CLI?** Each provider ships a small command-line tool (a CLI) that handles its local sign-in. Syndicate checks whether the CLI is installed, installs it for you if it's missing, and reads the provider's sign-in state to confirm the provider is ready. You don't need to type any commands yourself.
</Note>

## Connect Claude

{/* Video: Claude CLI install/check, browser auth approval, code paste if prompted, then Connected. */}

<Steps>
  <Step title="Open provider setup">
    On first launch, Syndicate opens provider setup automatically. To return later, open **Settings**.
  </Step>
  <Step title="Check Claude">
    Click **Check** on the Claude card. Syndicate inspects whether the Claude Code CLI is installed and signed in on this machine.
  </Step>
  <Step title="Install the CLI if needed">
    If the card shows **Not installed**, click **Install CLI**. Syndicate installs the Claude Code CLI using its managed Node and npm.
  </Step>
  <Step title="Sign in with your browser">
    If the card shows **Not signed in**, click **Connect**. Approve Claude in your browser, then paste the authentication code back into Syndicate if prompted.
  </Step>
  <Step title="Confirm Connected">
    The Claude card switches to **Connected** and lists available models.
  </Step>
</Steps>

## Connect OpenAI/Codex

{/* Video: Codex CLI install/check, browser ChatGPT approval, capability check, then Connected. */}

<Steps>
  <Step title="Open provider setup">
    On first launch, Syndicate opens provider setup automatically. To return later, open **Settings**.
  </Step>
  <Step title="Check OpenAI/Codex">
    Click **Check** on the OpenAI/Codex card. Syndicate inspects whether the Codex CLI is installed and signed in.
  </Step>
  <Step title="Install the CLI if needed">
    If the card shows **Not installed**, click **Install CLI**. Syndicate installs the Codex CLI using its managed Node and npm.
  </Step>
  <Step title="Approve in your browser">
    Click **Connect** and approve the Codex login in your browser with the ChatGPT account Syndicate should use.
  </Step>
  <Step title="Confirm Connected">
    The card switches to **Connected** after Syndicate confirms the account can actually run Codex.
  </Step>
</Steps>

<Warning>
Signing in alone isn't enough. Syndicate also checks whether the account can actually run Codex. Plan limits, quota, workspace policy, or expired credentials can still block provider readiness.
</Warning>

## Connect Gemini

{/* Video: Gemini CLI install/check, in-app terminal, Google browser approval, terminal completion, then Connected. */}

<Steps>
  <Step title="Open provider setup">
    On first launch, Syndicate opens provider setup automatically. To return later, open **Settings**.
  </Step>
  <Step title="Check Gemini">
    Click **Check** on the Gemini card. Syndicate inspects whether the Gemini CLI is installed and signed in.
  </Step>
  <Step title="Install the CLI if needed">
    If the card shows **Not installed**, click **Install CLI**. Syndicate installs the Gemini CLI using its managed Node and npm.
  </Step>
  <Step title="Follow the in-app terminal">
    Click **Connect**. Syndicate opens a small in-app terminal that runs the Gemini CLI for you. Follow its prompts, then approve the Google sign-in in your browser when it opens.
  </Step>
  <Step title="Or use a Gemini API key">
    Alternatively, paste a Gemini API key in **Settings** to skip the browser sign-in entirely.
  </Step>
  <Step title="Confirm Connected">
    The Gemini card switches to **Connected** and lists available models.
  </Step>
</Steps>

<Note>
Provider credentials stay on this machine. On a new computer, check and connect each provider again.
</Note>

## Done when

{/* Screenshot: connected provider with model list available. */}

- The provider card shows **Connected**.
- The provider has available models.
- New agents can select the provider's models during creation.

For what each provider status means and what to do next, see the [Status reference](/reference/providers#status-reference).

## Troubleshooting

- **Install failed.** Click **Install CLI** again — Syndicate retries with its managed Node and npm. If the second attempt also fails, check this machine's internet connection.
- **Card stays at "Not signed in" after you approved in the browser.** Click **Check** to refresh the card. If the status doesn't change, restart Syndicate and run **Connect** again.

For deeper details on each provider's sign-in behavior, see [Auth flows](/reference/auth-flows).
