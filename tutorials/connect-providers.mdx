Providers are what let agents actually run. Syndicate connects to Claude, OpenAI/Codex, and Gemini through their local CLIs, then verifies that the CLI is installed, signed in, and able to run.

You only need one connected provider to start, but connecting more gives each agent better model options.

## Open provider setup

{/* Screenshot: first-run provider gate or Settings > Providers with provider cards visible. */}

<Steps>
  <Step title="Open provider setup">
    On first launch, Syndicate opens the provider setup gate automatically. Later, you can return to the same setup from Settings.
  </Step>
  <Step title="Check a provider">
    Click **Check** so Syndicate can detect whether that provider's CLI is installed and authenticated on this machine.
  </Step>
  <Step title="Install the CLI if needed">
    If the card says **Not installed**, click **Install CLI**. Syndicate installs the provider CLI with its managed Node/npm setup.
  </Step>
  <Step title="Connect the provider">
    When the CLI is installed but not signed in, click **Connect** and complete the provider's auth flow.
  </Step>
</Steps>

<Note>
Provider credentials are local to the machine. If you set up Syndicate on another computer, check and connect providers again there.
</Note>

## Claude

{/* Video: Claude CLI install/check, browser auth approval, code paste if prompted, then Connected. */}

Claude uses the Claude Code CLI.

1. Click **Check** on the Claude card.
2. If Claude is not installed, click **Install CLI**.
3. Click **Connect**.
4. Approve the Claude sign-in flow in the browser.
5. Paste the authentication code into Syndicate if prompted.
6. Wait for Claude to show **Connected**.

Syndicate verifies Claude by checking the CLI and saved Claude credentials.

## OpenAI/Codex

{/* Video: Codex CLI install/check, browser ChatGPT approval, capability check, then Connected. */}

OpenAI uses the Codex CLI.

1. Click **Check** on the OpenAI/Codex card.
2. If Codex is not installed, click **Install CLI**.
3. Click **Connect**.
4. Approve the Codex login in the browser with the ChatGPT account you want Syndicate to use.
5. Return to Syndicate and wait for the readiness check.

<Warning>
Signing in is not always enough. Syndicate also checks whether the account can execute Codex. Plan limits, quota, workspace policy, or expired credentials can still block provider readiness.
</Warning>

## Gemini

{/* Video: Gemini CLI install/check, in-app terminal, Google browser approval, terminal completion, then Connected. */}

Gemini uses the Gemini CLI.

1. Click **Check** on the Gemini card.
2. If Gemini is not installed, click **Install CLI**.
3. Click **Connect**.
4. Follow the in-app terminal prompt.
5. Approve the Google sign-in flow in the browser when it opens.
6. Return to Syndicate and wait for Gemini to show **Connected**.

Gemini can also be considered connected when a Gemini API key is set in settings.

## Verify setup

{/* Screenshot: connected provider with model list available. */}

Provider setup is complete when:

- The provider card shows **Connected**.
- The provider has available models.
- New agents can select that provider's models during creation.

## What the statuses mean

| Status | What it means | What to do |
| --- | --- | --- |
| **Not checked** | Syndicate has not inspected this provider yet | Click **Check** |
| **Not installed** | The provider CLI is missing on this machine | Click **Install CLI** |
| **Not signed in** | The CLI exists, but credentials are missing or expired | Click **Connect** and approve auth |
| **Connected** | The CLI and credentials are ready | Use the provider in teams or agents |
