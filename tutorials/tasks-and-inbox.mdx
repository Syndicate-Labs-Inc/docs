Tasks show what the team is doing. Inbox messages show what agents are saying to each other — and to you. Together they are how you stay on top of a team without micromanaging it.

## Read the task board

{/* Screenshot: task board showing the four statuses. */}

Tasks live on a team's task surface. Every task has a title, an assigned agent, and a status.

| Status | What it means | What to do |
| --- | --- | --- |
| **Todo** | Queued or planned | Wait or reassign |
| **In Progress** | An agent is actively working | Monitor; don't pile on |
| **Blocked** | Work cannot continue | Read the reason and unblock |
| **Done** | Work is complete | Read the summary |

A completed task can include a summary of what the agent did, edited, or decided. Read summaries before queuing the next task — they often answer questions before you ask.

## Create or update a task

{/* Video: create a manual task and watch dispatch start. */}

You can:

- **Create a task manually** — assign a title, description, and agent.
- **Let dispatch create one** — when you send work or the Manager delegates, Syndicate creates a task automatically so you have a trace.
- **Mark a task blocked** — set status to **Blocked** and add a reason so the Manager knows it needs help.

## Read inbox messages

{/* Screenshot: inbox panel with mixed agent-to-agent and user-attention messages. */}

Agents send each other inbox messages to delegate, review, report back, or escalate. The inbox bell shows unread counts per team and across teams.

Open the inbox to:

- See what the Manager is asking specialists to do.
- Catch handoffs between agents.
- Spot questions that need you.

## Handle messages that need you

When an agent needs your input, it routes the request through the Manager or surfaces it as a user-attention item. These are decision points — the team is waiting on you before it can continue cleanly.

<Steps>
  <Step title="Open the user-attention item">
    Click into the message from the inbox or the activity surface.
  </Step>
  <Step title="Read the context">
    Skim the recent Manager messages so your answer matches what the team has already tried.
  </Step>
  <Step title="Reply with a clear instruction">
    Answer the question or pick the next step. Vague answers cause the loop to repeat.
  </Step>
  <Step title="Watch dispatch resume">
    Once you reply, the Manager dispatch resumes and the next task moves.
  </Step>
</Steps>

## When the team stalls

If unread inbox messages pile up or tasks all show Blocked:

1. Open the inbox and read the latest Manager messages first.
2. Stop any agent doing the wrong work.
3. Give the Manager one clear next instruction.
4. If the team is overrun, hard pause and reset the plan.
