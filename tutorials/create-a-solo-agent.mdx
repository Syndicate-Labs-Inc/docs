A solo agent is a one-on-one workspace with a single specialist and no Manager. Use it when you want fast back-and-forth on focused work.

## Before you start

You need:

- An approved Syndicate account.
- At least one provider connected — see [Connect providers](/tutorials/connect-providers).
- A local project folder for the agent to work in.

## Create the solo workspace

{/* Video: Solo Agent flow from click to first chat. */}

<Steps>
  <Step title="Start workspace creation">
    Open the workspace creation flow and choose **Solo Agent**.
  </Step>
  <Step title="Pick a specialization">
    Choose the specialist that matches the work — for example Frontend Developer, Technical Writer, QA Engineer, Researcher, or Backend Developer.
  </Step>
  <Step title="Pick a model">
    Select the [provider and model](/features/providers-and-models) the agent should run on. You can change this later in agent settings.
  </Step>
  <Step title="Select a project folder">
    Choose the local folder the agent should read, edit, and run commands in.
  </Step>
  <Step title="Open the workspace">
    Finish setup and land in the new solo workspace.
  </Step>
</Steps>

## Send the first message

{/* Screenshot: solo chat after the first message. */}

Ask the agent to do something small that matches its specialty. Good first messages include:

- Summarize what the project does.
- Identify the most important files.
- Suggest one improvement and explain why.

A short first task gives you a quick read on whether the specialization, model, and folder are wired up correctly.

## Where solo agents live

Solo workspaces show up under **Solo Agents** in the sidebar, separate from collaborative teams. Each one is a workspace with exactly one agent and no Manager.

## When to graduate to a team

A solo agent is best for focus. Switch to a team when:

- The work splits into parallel tracks.
- You need a Manager to coordinate specialists.
- You want review or QA after implementation.
- Multiple agents need to hand work back and forth.
