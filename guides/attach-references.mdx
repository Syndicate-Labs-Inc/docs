---
title: "Attach references"
description: "Give agents the project files and documents they should always have in mind."
---

References are read-only files and folders injected into an agent's context on every run. Use them for the material an agent should never have to go looking for. Concepts and limits: [References and context](/features/references-and-context).

## Pick the right scope

<Steps>
  <Step title="Decide who needs it">
    - Everyone, in every workspace (style guide, company standards): **Global**, managed in Settings.
    - Everyone in one team (project README, schema, requirements doc): **Team**, managed in team settings.
    - One agent (its role-specific source material): **Agent**, managed in its configuration panel.
  </Step>
  <Step title="Attach it">
    Click **+** next to the references section at that scope, then pick files or folders in the native picker.
  </Step>
  <Step title="Verify it landed">
    Click the reference to open its read-only preview. Then ask the agent something only the reference answers ("What does our style guide say about headings?") and confirm it knows.
  </Step>
</Steps>

## What gets injected

- **Files**: content up to 64 KB per file, 200 KB total per run. Bigger files contribute path and metadata only.
- **Folders**: a manifest of names and sizes, not contents.
- **PDFs**: extracted text (sources up to 10 MB).

## Keep it lean

- Reference the few documents that define the work; agents can read everything else in the working directory with their own tools.
- Audit occasionally with the agent's **Prompt Analytics** (configuration panel, Context): if references dominate the prompt and the agent rarely uses them, cut some.
- Prefer attaching one folder manifest over twenty individual files when the agent mostly needs to know what exists.
