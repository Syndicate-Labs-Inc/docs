Tasks show what the team is doing. Inbox messages show what agents are saying to each other and to you. Together they help you manage a team without micromanaging every specialist.

{/* Hero media: 20-30 second clip showing Atlas selected in Team Workspace, User Tasks open, an Agent Tasks item moving to Done, Agent Comms receiving an update, and Activity in the bottom tray. */}

## Open the Atlas task tabs

Select Atlas in **Team Workspace** to use the manager operations tabs:

- **User Tasks** for decisions, approvals, blockers, and interruption prompts that need you.
- **Agent Tasks** for tracked work assigned to Atlas or specialists.
- **Task Health** for risk, blocked work, missing owners, and weak task definitions.

Use these tabs instead of looking for approval cards in chat.

## Respond to a User Task

When Atlas needs your input, Syndicate creates a **User Task**. The team may soft pause while the item is open.

<Steps>
  <Step title="Open User Tasks">
    Select Atlas, then open **User Tasks** in the right rail.
  </Step>
  <Step title="Read the impact and source">
    Check whether the item is **Blocking**, **High**, **Medium**, or **Low** impact. Review the requester, related task, and description.
  </Step>
  <Step title="Reply, resolve, allow, or dismiss">
    Use **Reply** to answer Atlas, **Resolve** to close the item without a reply, or **Allow** / **Dismiss** for an Atlas interruption request.
  </Step>
  <Step title="Watch Atlas continue">
    After you answer or resolve the item, Atlas receives the update and can continue, relay your answer, or ask a follow-up.
  </Step>
</Steps>

## Review Agent Tasks

Open **Agent Tasks** to see tracked work. Each task can show its assignee, status, risk, phase, age, update time, and whether it needs human review.

Use **Cancel** when a task should not continue. Use the bulk abort action when a visible group of tasks needs to stop. Atlas receives a receipt so it knows not to recreate the canceled work unless you ask.

## Check Task Health

Open **Task Health** when the team feels stuck or risky.

Task Health can flag:

- Blocked tasks.
- Tasks requiring human decisions.
- High-risk or medium-risk tasks.
- Unassigned tasks.
- Tasks missing a deliverable, acceptance criteria, or done condition.

Click **Nudge Manager** on a specific item when you want Atlas to address that risk.

## Read Agent Comms

**Agent Comms** shows inter-agent dispatch messages. It is separate from your direct chat so coordination traffic does not bury the conversation you have with the selected agent.

For a specialist, **Agent Comms** shows messages sent to and from that specialist. For Atlas, it shows the broader team dispatch stream.

## Use the bottom tray

The bottom tray gives you operational context while tasks move:

- **Activity** shows recent tool, command, context, and message events for the selected agent or the team.
- **Files** lets you browse the working directory and open previewable files.
- **Uploads** lists files attached through team chat.

Use **Files** when you need to inspect project output. Use **Uploads** when you need to find a file or image you attached earlier.

## When the team stalls

If unread messages pile up or tasks all show **Blocked**:

1. Open **User Tasks** and resolve anything blocking.
2. Open **Agent Comms** to see the latest Atlas and specialist messages.
3. Check **Task Health** for risk or missing task details.
4. Stop any active agent doing the wrong work.
5. Give Atlas one clear next instruction.
6. Use **Stop Team** if the whole team needs a hard pause.
