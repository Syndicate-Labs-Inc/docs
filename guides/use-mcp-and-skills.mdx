Skills give agents better procedures; MCP servers give them reach into external systems. This guide wires up both on a working agent. Concepts: [Skills](/features/skills) and [MCP servers](/features/mcp).

## Add an MCP server and give it to an agent

<Steps>
  <Step title="Install the server globally">
    Open **Settings, MCP**. Use the **Catalog** for one-click presets (presets that need credentials ask for them first), or **Add a server** manually: pick the transport (Local command, Remote URL, or Remote SSE), name it, describe what agents use it for, add secrets if needed, and click **Add server**. Syndicate validates it with a live handshake before saving.
  </Step>
  <Step title="Check its health">
    The server row shows a health dot. Click **Test** to re-run the handshake and list the tools it exposes.
  </Step>
  <Step title="Enable it on the agent">
    Open the agent's configuration panel, **Tools (MCP)**, and flip the toggle for that server. Only this agent gains access; teammates are unaffected.
  </Step>
  <Step title="Verify in chat">
    Ask the agent to use the tool ("List my open GitHub PRs"). Under **Ask first** permissions you will get a Tool Request prompt; allow it and confirm the result.
  </Step>
</Steps>

## Use skills

Skills arrive with the agent's specialization; nothing to install for built-ins.

1. Type **/** in the agent's chat to open the slash menu and see everything it carries.
2. Pick a command, add context, send. The agent follows the skill's procedure.
3. For workflows none of your agents have, generate a package with [Agent Maker](/guides/agent-marketplace/agent-maker) or import one from the [community](/guides/agent-marketplace/community-agents).

## Patterns that work

- **Researcher + web/GitHub MCP**: `/investigate` becomes dramatically stronger when the agent can actually fetch sources.
- **DevOps + CI MCP**: `/deploy-review` and `/incident-response` with live pipeline visibility.
- **Keep tools role-scoped**: install globally, enable per agent, and give each agent only what its job needs. Fewer tools means better tool choices and a smaller blast radius.
