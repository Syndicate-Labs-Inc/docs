The team creation wizard ("Create a New Team") offers three paths. This page walks through each, plus the roster review and folder selection that follow.

## Path 1: Starter Templates

"Deploy a fully configured team in under 2 minutes. You can customize everything after."

Pick one of six pre-built teams; each loads a complete roster with roles, models, and skills already chosen:

| Template | What it is for |
|---|---|
| **Application Developers** | Full-stack development: architecture, frontend, backend, testing, and security |
| **Code Reviewers** | Automated PR reviews, linting, and quality gates |
| **Content Pipeline** | Research, write, edit, and publish content end-to-end |
| **Research Analysts** | Deep-dive analysis with multi-source synthesis |
| **DevOps Watchtower** | Monitor deployments, catch failures, manage infrastructure |
| **Customer Support Triage** | Route, respond, and escalate support tickets intelligently |

Full rosters, agent by agent: [Team templates reference](/reference/team-templates).

## Path 2: Generate Team

"Describe your project and let Manager pick specialized agents."

1. Choose a **Team model**: it generates the roster and becomes every agent's default model (you can still change individual agents in Review).
2. Write a **Project Description**, for example: "I'm building a full-stack SaaS app with React frontend, Node.js backend, PostgreSQL database. Need user auth, payment integration, and a dashboard..."
3. Click **Generate Team** (or press Ctrl+Enter). While it works you will see "Analyzing your project and assembling the ideal team...", then the proposed roster lands in Review.

## Path 3: Start from Scratch

Two sub-options:

- **Build Your Team**: "Starts with a Manager agent — add specialists until your roster feels right." You land in Review with just the Manager and add agents one by one.
- **Solo Agent**: "One specialist, no Manager. A direct 1:1 conversation." This branches into the [solo agent flow](/guides/create-a-solo-agent).

## The Review screen

Whatever path you took, you finish on **Your Team Roster**:

- **Team Name**: edit freely; defaults come from the template or generation.
- **Manager row**: always present with a star icon, described as the "Auto-created coordinator that delegates tasks to other agents". You can change its model but not remove it.
- **Agent rows**: checkbox to include or exclude, **Edit** to change name, specialization, or model, and **Skills (N)** to expand the baked-in skill list.
- **+ Add Agent**: opens a modal where you name the agent, pick a **Source** (syndicate, User Made, or Community), choose a **Specialization** (or "General-Purpose (no baked skills)"), and select a model.

At least one agent must be selected ("Select at least one agent."). Click **Create Team**.

## Working Directory

Last step: "Choose where your team's project files live." Browse to a folder and confirm. This is where agents read, write, and run commands; you can change it later in Team Settings.

## After creation

- Set the team's **Interruption Level** in Team Settings to control how often Atlas checks in with you (default is fine to start). See [Working with Atlas](/guides/create-your-first-team/atlas-orchestrator#interruption-level).
- If there are documents the team should know that live outside the folder, attach them in chat; they land in the bottom tray's **Uploads** tab.
- Send Atlas the first goal.

## Validation and limits

- Every agent needs a model from a connected provider. With no providers connected, the wizard warns: "Connect a provider in Settings before generating a team."
- If a selected model becomes unavailable (provider disconnected), the wizard resolves it to the first available model automatically.
- Your account tier caps teams, solo workspaces, and agents per team; see Settings, Account.
