Create a team when work needs coordination across a Manager and specialists. A team is a workspace tied to one local project folder, with the Manager coordinating the specialists that work inside it.

Use [Quick start](/guides/create-your-first-team/quick-start) when you want the shortest path. Use this guide when you want to understand the setup choices before you create or revise a team.

## Choose a setup path

Syndicate gives you three ways to create a team.

| Path | Use when | What you decide |
| --- | --- | --- |
| **Template** | You want a proven team shape for a common workflow. | Pick the template, confirm the project folder, and review the generated roster. |
| **Guided description** | You know the project and outcome, but want Syndicate to suggest the roster. | Describe the project, review the proposed Manager and specialists, then edit before creating. |
| **Build Your Team** | You know exactly which agents you want. | Start with a Manager, add specialists, and configure each role yourself. |

Start with **Template** when you are new to Syndicate. Use **Guided description** when the project has a clear goal but the team shape is not obvious. Use **Build Your Team** when you already know the roles you need.

## Pick the project folder

Every team needs a working folder. Agents use this folder when they inspect files, run commands, edit code, and gather project context.

Pick the smallest folder that contains the project. Avoid a parent folder that contains unrelated repositories, private notes, or files the team does not need.

## Review the roster

Before you create the team, review:

- **Manager**: the coordinating agent that plans, delegates, and follows up.
- **Specialists**: the agents that handle focused work, such as coding, review, writing, or research.
- **Names and roles**: labels that make handoffs and task ownership easy to scan.
- **Instructions**: constraints each agent should follow while working.
- **Provider and model choices**: the provider each agent uses for runs.

Keep the first roster small. You can add specialists later if the work needs more coverage.

## Choose a control mode

Control mode decides when the Manager must ask before acting.

| Mode | Best for | What to expect |
| --- | --- | --- |
| **Autopilot** | Trusted teams and routine work. | The Manager plans, delegates, and follows up without routine approval. |
| **Approve New Work** | Most new teams. | The Manager coordinates current work, but asks before expanding scope. |
| **Approve Each Step** | Sensitive projects or first-time setup. | The Manager asks before each delegation or follow-up. |

Start with **Approve New Work** unless you already know the team and project are low risk.

## After creation

When the team opens:

1. Send one small inspection task to the Manager.
2. Check that a task appears or updates.
3. Watch whether a specialist moves out of idle state.
4. Open the inbox if the team asks for input.
5. Review the first result before assigning implementation work.
