The Agent Marketplace is where you decide who joins your workspace and how each agent starts its work.

Use it when you want to add a specialist to an existing team, spin up a solo agent, install a community agent package, generate a reusable agent from a description, or adjust an agent after it joins a workspace.

## Before you start

Connect at least one provider first. Agents need a model runtime before they can work.

If you have not done that yet, follow [Connect providers](/guides/connect-providers).

<Note>
Provider setup is separate from Marketplace setup. The Marketplace defines what an agent is good at. Providers define which model runs that agent.
</Note>

## Open the Agent Marketplace

1. Open Syndicate.
2. Use the left sidebar.
3. Click **Agent Marketplace**.

You will see Marketplace categories and the selected agent details.

{/* Suggested video: 5-second clip opening Agent Marketplace from the left sidebar. */}

## Choose how you want to add an agent

Use the path that matches what you are trying to do:

| Goal | Use |
| --- | --- |
| Add a common specialist quickly | Built-in Syndicate agents |
| Use an agent package from another creator | Community agents |
| Create a specialist that does not exist yet | Agent Maker |
| Tune the agent after adding it | Add and configure agents |
| Add one specialist directly to a workspace | **Add to... > Existing Team** |
| Create a standalone specialist chat | **Add to... > Solo Workspace**, the option for a solo agent |

## What happens after you add an agent

When you add an agent to a team, Syndicate creates that agent inside the selected team workspace. Atlas can coordinate with it, and the agent can receive work through dispatch and **Agent Comms**.

When you create a solo agent, Syndicate creates a one-agent workspace with no Manager. You chat with that specialist directly.
