Built-in Syndicate agents are the fastest way to build a useful team. They ship with Syndicate and are ready to add from the **Syndicate** category in the Agent Marketplace.

The Manager is itself a built-in Syndicate agent — it is the one you add when you want a coordinator to lead a [team](/features/teams). The other built-ins are specialists you add alongside the Manager (or run as solo agents).

## When to use a built-in agent

Use a built-in agent when you want a reliable specialist for a common workflow:

| Need | Good fit |
| --- | --- |
| Coordinate a team | Manager |
| Build UI | UI Designer or Frontend Developer |
| Build APIs, auth, or data flows | Backend Developer |
| Review code, docs, copy, or plans without editing | Review Agent |
| Write docs | Technical Writer |
| Test and find regressions | QA Engineer or Test Engineer |
| Research and compare options | Researcher |
| Plan infrastructure or releases | DevOps Engineer or Site Reliability Engineer |

## The Manager

The Manager is a built-in agent built to orchestrate a team — not to write or edit files itself. You give it an outcome. It decides which specialist on the team should do what, dispatches the work, tracks status as agents finish or get blocked, and follows up until the outcome is met.

A Manager comes preloaded with the roster context it needs to coordinate. That includes the names and specializations of the agents on its team, the skills and commands available to each one, the current status of work in progress, and the rules for how teammates hand work back and forth. You don't need to re-introduce the team every time you start a request. The Manager already knows who is on it and what each agent is good at.

Use a Manager when the work has multiple parts that benefit from coordination. If you only need one focused back-and-forth with one specialist, skip the Manager and use a [Solo Workspace](/features/solo-agents) instead. Each team has one Manager — if you need to change it, review the team configuration first.

For how the Manager dispatches work and what control modes are available, see [Teams](/features/teams).

## Add a built-in agent to a workspace

Open the Marketplace, select the agent in the **Syndicate** category, click **Add to...**, and choose either **Existing Team** or **Solo Workspace**. For the full flow and what happens after you add an agent, see [Choose how you want to add an agent](/guides/agent-marketplace#choose-how-you-want-to-add-an-agent).

## Skills on built-in agents

Most built-in specialists ship with a small set of skills. Skills are reusable workflow instructions bundled into an agent — they tell the agent when to use a workflow, what steps to follow, and what output to produce.

You can inspect or change the skills on a built-in agent in the agent configuration panel after you add it. See [Skills](/features/skills) for what skills do and when to add more.

## Configure the agent after adding it

Open the agent settings from the workspace to adjust:

- Model/provider.
- Context settings.
- Skills attached to the agent.
- MCP servers available to that agent.

<Tip>
Start with the default model if you are recording a quick demo. Change models later when you want to show provider flexibility.
</Tip>

## Limits & gotchas

- **Built-in agents still need a provider.** Each agent runs on the model and provider you set in its configuration. If you haven't connected a provider yet, follow [Connect providers](/guides/connect-providers) first.
- **Don't add a Manager just for one question.** For a single back-and-forth with one specialist, use a Solo Workspace instead — see [Solo agents](/features/solo-agents).
- **One Manager per team.** A team is built around a single Manager coordinating specialists. If you need to change the Manager, review the team configuration first.
- **Built-ins don't know your project until you tell them.** Add references, instructions, or a project folder so the agent has the context it needs — see [Attach references](/guides/attach-references).

## Verify it worked

After adding the agent:

1. Open the team or solo workspace.
2. Confirm the agent appears in the roster.
3. Send a small request.

Good first prompts:

- "Read this folder and summarize what is here."
- "Review this plan and list the top risks."
- "Draft a short README outline for this project."
