---
title: "Settings"
description: "Settings areas you commonly need while setting up Syndicate."
---

Settings is where you connect providers, choose models, configure MCP servers, and adjust team or agent behavior.

## Provider settings

Use provider settings to:

- Connect or reconnect Claude, OpenAI/Codex, and Gemini.
- Inspect provider readiness.
- Resolve authentication problems.
- Choose available models for agents.

## MCP settings

Use **MCP** settings to:

- Browse available MCP servers.
- Review setup requirements.
- Install or configure a server.
- Add a server manually.

After you configure an MCP server, enable it for the agents that need it in **Agent Settings**.

## Team settings

Open **Team Settings** from **Team Hub**, a team card, or the **Team Workspace** header.

Use team settings to:

- Rename teams.
- Relink the working directory.
- Set **Interruption Level**.
- Hard pause or resume the team.
- Delete the team.

## Agent settings

In a team, open **Agent Settings** from the right rail. For a solo agent, open the agent and use the settings entry point in the workspace header or right rail.

Use agent settings to:

- Review identity and specialization.
- Rename non-manager agents.
- Read the agent's **Soul** instructions.
- Change context settings.
- Review skills and slash commands.
- Enable or disable MCP servers for that agent.
- Delete a non-manager agent after confirmation.

Context settings control how much history, **Agent Comms** context, tasks, references, team context, and connector context the agent receives during runs. In a team, these settings can include team-only context such as tasks, references, team context, and **Agent Comms**. For a solo agent, they apply to the solo agent's own chat, files, and enabled connectors. The context editor includes quick presets, lookback limits, reference limits, token budget, included sections, and prompt analytics.
