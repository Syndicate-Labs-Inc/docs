Short answers to the questions that come up most often. Each entry links to the deeper page.

## What is Syndicate?

Syndicate is a desktop app for running AI agents against your local project folders. You can work with a single specialist or an Atlas-led team. See [Feature overview](/features/overview).

## Do I need beta access to use Syndicate?

Yes. Access is currently invite-based. Request access from the Syndicate site and create your account against the approved email. See [Install and sign in](/guides/install-and-sign-in).

## Which providers are supported?

Syndicate supports three providers:

- **Claude** through the Claude Code CLI.
- **OpenAI/Codex** through the Codex CLI.
- **Gemini** through the Gemini CLI.

See [Connect providers](/guides/connect-providers).

## Where are my credentials stored?

Provider credentials are stored locally on the machine that connected them. Syndicate does not sync provider credentials between machines. Connect providers once per machine. See [Auth flows](/reference/auth-flows).

## Where does my workspace data live?

Workspace data, including teams, agents, messages, tasks, references, and uploads, is stored locally on your machine and scoped to your signed-in user. Syndicate uses your account to sign you in and check beta access.

## Can I use Syndicate offline?

Agent runs need the provider you chose to be reachable, so they require internet. Sign-in and account-related checks also need connectivity.

## What is the difference between a team and a solo agent?

- A **team** has Atlas plus specialists. Atlas plans, delegates, routes follow-ups, and creates **User Tasks** when it needs you.
- A **solo agent** is a single specialist with no Manager, no **Agent Comms**, and no inter-agent dispatch.

See [Workspaces](/features/workspaces).

## How does Interruption Level work?

**Interruption Level** controls when Atlas asks before continuing.

- **Minimal** interrupts only for true blockers, explicit user attention, missing access, high-risk work, or required approval.
- **Balanced** interrupts for unclear scope, meaningful new workstreams, medium or high risk, or completion approvals.
- **Hands-on** interrupts before major assignments, broad follow-ups, risky decisions, and completion reviews.
- **Manual** turns most delegations, follow-ups, and re-routes into **User Task** approval prompts.

See [Teams](/features/teams).

## What are User Tasks?

**User Tasks** are decisions, approvals, blockers, and interruption prompts that need you. They replace chat approval cards. Select Atlas in **Team Workspace**, then open **User Tasks**.

See [Tasks](/features/tasks).

## How does Syndicate choose which model to run?

Each agent has a provider and model. When an agent does not specify one, Syndicate uses your defaults. See [Providers and models](/features/providers-and-models).

## What is an MCP server?

An MCP server is an external tool integration. Once configured and enabled for an agent, it gives that agent access to a service like file search, browser automation, documents, or another API. MCP server credentials are local to the machine and the provider runtime. See [MCP servers](/features/mcp).

## How do references work?

References are files or folders you attach so agents can use them as context. They can be attached at global, team, or agent scope. Syndicate manages context size and may trim references when a run grows too large. See [Attach references](/guides/attach-references).

## How do I stop a runaway team?

Stop the active agent response, use **Stop Team** to hard pause the team, then adjust instructions, references, or **Interruption Level** before resuming. See [Troubleshooting workspaces and dispatch](/reference/troubleshooting-workspaces).

## Can I use the same account on more than one machine?

You can sign in to Syndicate from more than one machine. Provider credentials are local to each machine, so you need to connect providers separately wherever you sign in. Workspace data is also stored locally per machine.

## How does Syndicate update?

Syndicate is a packaged desktop app and surfaces an update prompt when a new release is available. Apply the update from within the app when you see it.

## Where do I get help?

Email [support@usesyndicate.org](mailto:support@usesyndicate.org). For setup problems, start with [Troubleshooting](/reference/troubleshooting) or [Troubleshooting workspaces and dispatch](/reference/troubleshooting-workspaces).
